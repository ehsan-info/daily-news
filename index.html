Var, Let, and Const - What's the Difference?

VAR: When 'var' is declared outside function it can be accessed or used in whole window. But if it is declared between a fucntion then it can not be accessed outside that function.

'var' can be re-declare and updated. This means that if we want to change the value of var variable or re-declare the variable it will not produce an error.

Let: It is an improved version of 'var'. It solved the problems of 'var'

Anything bounded by {} is considered as a block. Any variable within this block can only be accessed within that block. We can not use outside that block

Let can be updated within its scope like var but can not be re-declared. If we try to re-declare, then it will give an error.

Const: Like 'let' declarations, 'const' can also be used within the scope. But here it can not be re-declared and not even updated the value. In both cases it will give an error.

Differences Between Arrow Functions and Regular Functions

Arrow function is a new feature of ES6 that is more concise syntax for writing function expressions.Though they both serve a similar role but there are some difference between them

Developer now can get the similar result by doing less code of lines.

Arrow functions dont have their own 'this' like regular function. The value of 'this' will be same throughout the life cycle of the function.

Regular functions declaration or expressions are callable or constructurable but arrow functions are only callable not constructurable.

Why do we use Template String ?

Template Strings are very powerfull feature of ES6. It has saved us from hassle we used to do it before. By using this we do not need to concatenate string ar variables like in older version of JavaScript. It lets us insert or interpolate expression and variables intro string without using concatenation. To do this we just need to use backtick and it will delimit them.

Another great feature of Template String is that we can write multi lines strings for better code readability. But in old version we had to concatenate each line of string to make multiline code.
